# Hider 项目计划清单

> 分类说明：
> - **待优化**：已有功能存在问题，需要改进（标注当前问题 + 未来期望）
> - **待开发**：尚未实现的新功能（标注未来期望）
> - **优先级**：P0 = 必须尽快完成 / P1 = 重要但不紧急 / P2 = 可延后
> - **定位**：🆓 免费版基础质量 / 💎 Pro 买断版功能

---

## 一、待优化（免费版 - 基础质量）

### 1. ~~网页阅读体验~~ 🆓 `P0` ✅

**已完成：**
- 实现阅读模式（Reader Mode）：通过 CSS 注入净化页面，移除广告/侧栏/悬浮层
- 文件模式的字体大小、行高、颜色设置在阅读模式下同步生效
- CSS 在页面导航时自动重新注入，确保持续生效

**遗留：**
- 评估是否迁移到 `WebContentsView`（Electron 推荐方案）

---

### 2. ~~EPUB 解析健壮性~~ 🆓 `P1` ✅

**已完成：**
- OPF 解析改用逐属性提取，兼容任意属性顺序
- 支持 URL 编码路径、`..` 相对路径解析
- 提取章节标题（h1/h2/h3/title）
- 增强 `htmlToText()`：支持更多 HTML 实体、列表符号、智能引号
- 每章独立错误处理，解析失败优雅降级

**遗留：**
- 不支持 EPUB 内嵌图片、CSS 样式
- 没有章节目录导航，所有章节拼成一个长文本

---

### 3. ~~代码架构集中化~~ 🆓 `P1` ✅

**已完成：**
- `app.js` 拆分为 ES Module 架构：
  - `modules/state.js` — 共享状态 + DOM 引用 + 工具函数
  - `modules/content.js` — 内容显示、模式切换、书签、阅读模式
  - `modules/settings.js` — 设置面板、预设、快捷键录制
  - `app.js` — 入口文件，IPC 事件、拖拽、鼠标追踪、窗口拖动
- 无循环依赖，DAG 结构：state ← content ← settings ← app

### 5. 悬停模式与窗口交互状态机 🆓 `P0`

**已完成：**
- 将悬停模式从零散事件监听重构为独立状态机
- 修复“悬停模式只变淡、不做鼠标穿透”的核心问题
- 接入 Electron `setIgnoreMouseEvents`，实现 ghost / interactive 两态切换
- 将悬停命中检测迁移到主进程轮询，避免窗口退场后渲染层定时器不可靠
- 将 hover 模式回归为纯显隐 + 鼠标穿透，不再擅自提升窗口层级
- 明确区分 hover 与 pin：hover 控制显隐，pin 控制是否持续置顶
- 新增 titlebar 钉子按钮，和设置页 / 托盘菜单的置顶状态同步
- 收紧 hover 强制交互条件，避免设置面板/书签/输入焦点偶发阻止鼠标离开后自动隐藏
- 将设置面板、书签弹层、输入聚焦、窗口拖拽纳入统一交互锁，避免可见但不可点
- 设置应用后立即同步 hover 状态，减少 UI 状态与窗口状态断链
- 修复 `update-hover-window` IPC 返回不可序列化 timer 对象导致的运行时断链
- 已完成真实 Electron GUI 回归：移出窗口后让出前景，移回窗口后恢复显示
- 已完成真实 Electron GUI 回归：未开启置顶时，切到其他应用后 Hider 不再持续压在最上层

**遗留：**
- 继续补充实际 GUI 手动回归，覆盖文件模式 / 网页模式 / 文本模式 / 老板键组合场景
- 评估是否继续将老板键、文本模式、悬停模式统一为更完整的窗口行为状态机

## 二、待开发（Pro 买断版功能）

> 商业模式参考：一次性买断 + 12 个月功能更新，大版本升级可折扣
> 参考产品：Witch（永久许可 + 至少 1 年更新）、Koodo Reader（本地同步 vs 云同步分层）、Obsidian（本地免费 + 同步收费）

---

### 1. ~~网页阅读增强引擎~~ 💎 `P0` — 最核心卖点 ✅（v1 已完成）

**已完成：**
- 站点适配与规则化网页阅读系统
- 16 个内置站点规则（起点、番茄、飞卢、纵横、晋江、微信公众号、知乎等）
- 分离控制文字/背景颜色覆盖（独立 overrideTextColor / overrideBgColor 开关）
- 按域名记忆站点配置（字体、颜色、隐藏图片、CSS 选择器）
- 自动阅读模式（按域名规则自动进入 Reader Mode）
- 用户可自定义内容选择器和移除选择器（高级选项）
- Pro 门控（CSS + 代码级别）

**遗留：**
- 正文提取智能化（通用 Readability 算法）
- 页内搜索、章节跳转
- 阅读位置稳定恢复

---

### 2. 隐身自动化 / 场景工作区 💎 `P0`

**未来期望：**
- **工作区配置**：将标题栏、窗口尺寸、透明度、热键、预设打包成一个「工作区」
- **一键切换**：工作模式 / 午休模式 / 单行模式 / 极简模式，一键切换整套配置
- **自动规则**：
  - 切到会议软件（Zoom/Teams/钉钉）时自动收起
  - 失焦 N 秒后自动隐藏
  - 不同网站或文件自动套用不同配置
- **增强老板键**：更强的隐身行为（清除最近文件记录、伪装窗口标题等）
- **快速恢复**：老板键触发后，一键恢复到上次阅读状态和位置

**核心价值：** 少一步操作，少一次暴露风险

---

### 3. 本地优先跨电脑同步 / 迁移 💎 `P1`

**未来期望：**
- **导出 / 导入**：一键导出完整配置包（设置、进度、书签、工作区）
- **加密备份**：支持加密的备份包，保护隐私
- **本地同步**：指定同步目录，利用用户自己的 iCloud / OneDrive / Dropbox / WebDAV 实现跨电脑同步
- **冲突合并**：多设备同步时的冲突检测与合并策略
- **一键迁移**：换电脑时一键恢复所有数据和配置

**核心价值：** 换电脑不丢进度，无需自建云服务，零持续成本

---

### 4. 高级阅读控制 💎 `P2`

**未来期望：**
- **焦点阅读**：段落高亮 + 上下文淡化，引导视线聚焦
- **统一书架**：文件阅读和网页阅读统一管理，最近阅读、收藏列表
- **统一搜索**：跨文件/书签的全文搜索
- **章节导航**：EPUB 章节目录、TXT 智能分章、锚点书签
- **键盘流**：完整的键盘快捷键体系，无需鼠标即可完成所有操作
- **多窗口 / 小浮窗**：支持多个阅读窗口或迷你浮窗模式
- **内容规则**：针对不同内容类型（小说、技术文档、漫画）的阅读规则

**核心价值：** 重度用户增强包，让高频使用更顺手

---

### 5. Pro 支付与激活系统 💎 `P1`

**架构设计：**
- **双通道并存**：激活码（赠送/推广）+ 在线付费（主付费通道）
- 激活码保留作为赠送/推广渠道（测试用户、合作博主）
- 主付费通道走在线付费直接解锁（付款后无需激活码）

**技术方案：**
- 设备指纹生成（Electron 端，基于硬件信息）
- Cloudflare Worker 验证 API（`/activate` 激活码 + `/verify` 状态查询 + `/pay-callback` 支付回调）
- D1 数据库存储设备绑定 + 会员状态
- 面包多/爱发电作为支付聚合（支持支付宝/微信，个人可用）
- 离线降级：上次验证结果缓存本地，7 天内有效
- 一码一机，防止激活码共享

**数据模型：**
```sql
devices (device_id TEXT PK, is_pro INT, license_key TEXT, order_id TEXT, created_at TEXT)
```

**用户流程：**
- 方式 A（激活码）：输入码 → 联网验证 + 绑定设备 → Pro
- 方式 B（付费）：点击升级 → 跳转付费页 → 付款回调 → 自动解锁

**渐进路径：**
1. 阶段 1：面包多 + 设备绑定激活码，验证付费意愿
2. 阶段 2（100+ 付费）：加在线付费直接解锁
3. 阶段 3（500+ 付费）：考虑订阅制或自建支付

## 三、明确不做（当前阶段）

| 方向 | 原因 |
|------|------|
| 官方云账号 / 自建云同步 | 持续服务器成本，不适合买断模式 |
| AI 功能（总结/OCR/语音/翻译） | 持续 API 成本，适合未来订阅制 |
| 在线服务依赖 | 增加运维负担，偏离本地优先定位 |
